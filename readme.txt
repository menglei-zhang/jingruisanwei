

首先先从最低级开始写起
   1.业务员逻辑
   订单管理模块
   业务员:订单提交
  业务员提交需求表单包括
订单编号,客户名称,订单金额,业务员名称，
交货日期(2019年4月8日,上午(8:30-11:30),下午(11:30 - 5:00),晚上(5:00-20:00))?
制作工艺,打磨要求,数量, 克数, 打印材料, 注意事项, 选填(文件名称)，图片(下面放图)和附件上传，
业务员确认生产部（3D打印/CNC机器/复模）（这边到底是业务员选择生产部还是提交订单的组长选择生产部）,
可以复选

业务员只能看到自己的订单信息，


2.组长逻辑

(1)首先是确定下面组员提交上来的表单 有确认和返回两个按钮
(2)然后组长可以看到自己组里所有人的订单信息
(3)组长提交时提交给生产部时按正常从上而下谁空安排谁
(4)组长拒绝订单返回给业务员

3.生产部 编程逻辑



编程逻辑

(1)生产部只能看到订单里的订单编号  业务员名称，
交货日期制作工艺,打磨要求,数量, 克数, 打印材料, 注意事项, 选填(文件名称)，图片(下面放图)和附件上传
(2)应该业务员提交的表单里上机机号,下机时间应该是空着的  这个应该是生产部填写
(3)生产部填写完订单后提交给手工
(4)编程不可拒绝订单

手工逻辑
(1)得到订单后可以拒绝返回给编程
(2)手工完成后将订单返还给业务员
